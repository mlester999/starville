Defaults, Asset Resolution, World Composer Integration, and Optional Admin Overrides. The local
hosted-validation repair is preserved separately and is not reclassified as Phase 12C visual work.

Phase 12C is **locally complete, hosted validation pending**: World Visual Overhaul, Map

## Local Phase 12C boundaries

- Shared machine-readable art, scale, camera, depth, shadow, quality, and validation policy. The
  bundled asset pipeline consumes the same tokens for projection, anchor, structure-scale,
  dimension, and animation-budget checks.
- Renderer framing, decorative world apron, terrain variation, grounding, depth, labels/bubbles, and
  performance-mode foundations.
- A 47-object, 36-collision repository-local Lantern Square draft with seating, landmark hierarchy,
  and edge clusters. It has zero shared-policy errors, zero warnings, and only the truthful
  temporary-art recommendation; no hosted published revision is changed.
- Compact expandable HUD, explicit loading/error states, responsive safe zones, accessible settings,
  and a real phone movement pad wired to the collision-safe runtime input.
- Exact-pin Draft Preview, exact-revision Visual Readiness, and shared-policy Composer diagnostics.
  They guide an administrator and do not automatically alter a draft or weaken publication controls.
- The existing protected no-progression Game Test mounts the production renderer after a valid
  grant. Its exact authorized revision is the default; authorized Lantern Square sessions may
  explicitly select the checked-in unpublished composition with verified bundled-only in-memory
  deliveries. It provides deterministic one/eleven-player, chat-bubble, depth, low-quality, Reduced
  Motion, and HUD-review controls.
- Owner-facing actual-renderer screenshots, 200-percent browser zoom, signed-in parity,
  mobile/desktop memory, GPU/frame timing, WAN timing, and eleven-network-client evidence remain
  unchecked until exercised in the required real environments.

Phase 12C does not add database authority, an animal/livestock system, or mechanics from Fablesol,
Pokentara, Sailana, AIvanza, or any previous project. It does not authorize a hosted write, uploaded
asset activation, world publication, migration push, deployment, commit, or Git push.

The phrase “hosted validation pending” covers the protected Admin/Game Test/revision/asset paths,
the separate Phase 12 hosted-validation repair, and the Phase 12C private manifest-contract
migration that accepts canonical furniture objects and optional quarter-turn rotation. Both
forward-only migrations remain local and unpushed. No hosted result is inferred from local code or
tests.
