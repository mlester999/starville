# Phase 8A owner acceptance checklist

1. [ ] Open two different browser sessions.
2. [ ] Connect two eligible wallets.
3. [ ] Join the same world and channel.
4. [ ] Confirm both players appear exactly once with safe nameplates.
5. [ ] Walk with Player A.
6. [ ] Confirm Player B sees authoritative walking/jogging and displacement-based facing; stop
       Player A and confirm idle begins while the last facing is retained.
7. [ ] Walk behind and in front of objects.
8. [ ] Confirm foot-position depth sorting.
9. [ ] Switch Player B to another channel.
10. [ ] Confirm the players disappear from each other.
11. [ ] Return Player B to the original channel.
12. [ ] Confirm presence returns without duplicates.
13. [ ] Refresh one client.
14. [ ] Confirm fresh-ticket snapshot recovery.
15. [ ] Hide and restore the browser tab.
16. [ ] Confirm resynchronization without duplicate presence.
17. [ ] Travel to another world.
18. [ ] Confirm old-world players are cleared.
19. [ ] Suspend one player and confirm bounded realtime disconnection.
20. [ ] Restore that player and reconnect with a fresh authorized session.
21. [ ] Activate maintenance and confirm new and existing realtime access is blocked.
22. [ ] Confirm authorized administrator population visibility and unauthorized denial.
23. [ ] Test 360×800, 390×844, 768×1024, and 820×1180 layouts.
24. [ ] Confirm keyboard focus, polite status announcements, and reduced-motion behavior.
25. [ ] Review the 10/20/40-player, multi-channel, movement-burst, and reconnect-burst load results.

Hosted two-wallet acceptance requires two private eligible wallet sessions and owner-approved
migration deployment. Never paste wallet secrets into output.
