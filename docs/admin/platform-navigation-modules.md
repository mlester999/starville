# Navigation and module registry

Navigation items reference stable registered route, module, and icon keys. Administrators may change
labels, order, groups, badges, and collapsed presentation. They cannot create arbitrary routes or
icons. Hidden navigation never grants or removes permission.

Authentication, administrator authorization, audit, security settings, and platform configuration
are required modules. Optional modules retain data, permissions, and audit history when disabled.
Dependencies are validated before publication; World Assets, for example, requires World Management.
Direct authorization remains server-authoritative.
