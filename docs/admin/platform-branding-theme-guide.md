# Branding and semantic theme guide

Branding supports game and administration names, tagline, description, support, copyright, website,
documentation, community, Discord, X, and legal links. Use plain text. Scripts, markup, controls,
embedded credentials, and insecure external destinations are rejected.

Images must complete the World Asset Manager lifecycle. Profiles are `brand_logo`, `brand_mark`,
`favicon`, `admin_login_background`, `landing_hero_background`, and `social_share_image`. Upload PNG
or WebP—not SVG—then process, validate, review, approve, activate, and select the exact active
version. Recommended sources are 512 × 512 for favicons, 1920 × 1080 for login/hero backgrounds, and
1200 × 630 for social sharing.

Themes use 18 semantic tokens rather than arbitrary CSS. Validation measures text and control
contrast at 4.5:1 and focus indication at 3:1. The editor can restore the Starville twilight or cozy
light preset before saving. Typography uses only `system_display`, `system_sans`, and `system_mono`;
external font URLs and raw font uploads are not supported.
