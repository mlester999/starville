# Platform content editor guide

Create a draft from the active version before editing. Each editor changes one structured section
and requires a reason. Unsaved edits display a warning. Saving uses the visible revision so a
concurrent edit returns a conflict instead of overwriting another administrator.

Admin-login settings change presentation only. Email/password fields, MFA messaging, recovery,
security warnings, and authorization behavior cannot be removed. Landing settings are structured;
each registered section exposes enabled state, unique order, bounded copy, paired CTA fields,
approved visual selection, and bounded supporting items. Raw HTML, arbitrary embeds, CSS,
JavaScript, and wallet-access overrides are rejected.

Preview the exact landing, admin shell, and admin-login draft beside the current landing at mobile,
tablet, and desktop sizes. Public applications remain on the active version until an authorized
reviewed publication succeeds.
